# 汇编语言程序的上机操作过程

## 所需环境
- windows(version7 or higher)
- Editor（VSCODE+8086ASM_PLUGIN）
- MASM.EXE （汇编程序）
- LINK.EXE（连接程序）
- DEBUG.EXE（调试程序，DOS 外部命令文件）

## 运行汇编语言程序的操作步骤
```
1.用编辑程序建立.ASM 源文件；
2.MASM.EXE把ASM 文件转换成.OBJ 文件；
3.LINK.EXE把OBJ 文件转换成.EXE 文件；
4.DOS环境直接输入文件名就可执行该程序。
```

## 汇编语言源程序上机操作过程
```
第一步：建立汇编源程序（即：建立文件名.asm）
    使用编辑器建立.asm后缀名的汇编源文件
第二步：汇编目标文件（即：编译为.obj .lst .crf 文件）
    DOS 提示符下键入MASM filename 回车。
    这时汇编程序的输出文件可以有三个（分别：.obj .lst .crf)
第三步：编译不通过,重新修改（错误类型：源程序语句不合法）
    在执行过第二步后，如有出错信息时，就不能跳跃到第四步去
    如果强行执行第四步，将无任何有效结果。
    依据行号进行修改并存盘，再次进行汇编
    直至第二步通过为止。便可继续向下执行第四步。
第四步：连接为可执行文件（即：连接为.exe 或.com 文件）
    格式：在DOS 提示符下： LINK filename 回车
第五步：运行编译好的可执行文件
    当第四步通过后，会产生一个可执行文件，这时只需运行这个程序。
    程序是否运行稳定、结果是否正确。如一切正常，便可进入第六步了。
第六步:用调试工具调试，重新修改（逻辑错误）
```
